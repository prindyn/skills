# Glossary

Web-design terms that come up often. When the user uses one of these and may not know what it means, define it briefly in plain language. When you use one, do the same.

## Brand identity
Colors, logos, fonts, shapes, and other aesthetics used to associate the brand with one consistent look. Closely related to but distinct from "brand" itself, which includes voice, values, and reputation.

## Brand kit / Design system
A documented set of standards and reusable components — colors, fonts, button styles, spacing, etc. — that guide every visual decision. The single source of truth so the site stays consistent as it grows. (See `templates/design-system.md`.)

## Click path
The sequence of links a visitor follows through the site, in order. Designers often plan a "happy path" — the click sequence that leads to the primary CTA — and then measure how many visitors actually follow it.

## Conversion rate
The percentage of visitors who complete a defined goal. Calculated as `(conversions / visitors) × 100`. Example: 100 visitors, 15 newsletter signups → 15% conversion rate. The exact goal varies (purchase, signup, booking, download).

## CSS (Cascading Style Sheets)
The language that controls how a webpage looks — colors, fonts, spacing, layout. Works alongside HTML (the structure) and JavaScript (the behavior). Most platform users never touch it directly; the platform generates it for you.

## CTA (Call to Action)
The specific action you want a visitor to take — "Buy now," "Book a call," "Sign up." Usually appears as a button. The most important element on most pages.

## Design elements
The visible building blocks of a webpage: buttons, images, lines, shapes, text, spacing. Each element should serve the page's purpose; if it doesn't, cut it.

## Hero
The first section visitors see at the top of a page (especially the homepage). Typically contains the headline, subhead, primary CTA, and a strong visual.

## HTML (HyperText Markup Language)
The language that defines the structure and content of a webpage — headings, paragraphs, links, images. Like CSS, most platform users never write it directly.

## Landing page
A single-page website (or single page within a site) focused on one specific conversion goal. Often used for ad campaigns, lead generation, or product launches.

## Responsive design
Design that adapts to any screen size — desktop, tablet, phone — without breaking layout or readability. Now a baseline expectation, not a feature.

## SEO (Search Engine Optimization)
The practice of structuring a site so search engines (mostly Google) can understand it and rank it for relevant searches. Includes content quality, keywords, page titles, meta descriptions, alt text, site speed, mobile-friendliness, and internal linking.

## SSL / HTTPS
The encrypted connection between the visitor's browser and your website. Indicated by the lock icon in the address bar. Required for any site collecting form data — and a baseline trust signal even if you're not.

## UI (User Interface)
The visible parts of the site that users interact with — buttons, forms, menus, the screens themselves. "What it looks like and how it behaves to the touch."

## UX (User Experience)
The felt experience of using the site — does it feel easy, slow, frustrating, delightful? Goes beyond UI to include speed, content quality, and whether the site actually helps the visitor accomplish their goal.

## Visual hierarchy
The arrangement of elements that signals their relative importance. Achieved through size, color, contrast, position, and whitespace. Tells the visitor's eye where to look first, second, third — usually subconsciously.

## WCAG (Web Content Accessibility Guidelines)
International standards for making websites accessible to people with disabilities. WCAG AA is the practical baseline most professional sites target. Includes contrast minimums, keyboard navigation, alt text, and more.

## Whitespace
The empty space around and between elements on a page. Not "wasted" space — it's how design breathes and how the eye finds what matters.

## Wireframe
A low-fidelity sketch or layout of a page showing where elements go, without final colors, fonts, or content. The architectural blueprint before the visual design.
